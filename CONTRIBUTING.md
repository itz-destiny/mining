
# Contributing

Thanks for your interest in contributing! Please follow these steps before opening a PR:

1. Fork the repository and create a feature branch.
2. Run the tests and linting locally (`pip install -r backend/requirements.txt`).
3. Keep changes focused and document them in the PR description.
4. Ensure no secrets are committed. Use `.env` or GitHub secrets for credentials.
5. For infrastructure changes, include deployment notes and rollback steps.

We welcome bug reports, documentation improvements, and code contributions. Thank you!
